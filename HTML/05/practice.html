Session 5: バグを直し、DevToolsを使う

壊れたHTMLを実際に直しながら、検証ツールの使い方を身につけます。

説明

HTMLは多少壊れていてもブラウザが補正して表示します。ただし補正に頼ると環境差で不具合が出やすくなるため、DevToolsやValidatorで原因を特定し、正しい構造に直す力が重要です。

やってほしいこと

  1. bug.html を開いて、表示崩れの原因を修正する
  2. DevTools ElementsでDOM構造を確認する
  3. Consoleにエラーがない状態にする
  4. W3C Validatorでチェックし、警告内容を読む
[ポイント]
  • ブラウザは多少の文法エラーを補正して表示する
  • ブラウザ補正は便利だが、実務では正しい文法での記述が前提
    • この後学ぶ、Reactでは文法が厳密にチェックされ補完されないようになっています
