15-1. 非同期JavaScript入門 デモ

同期処理と非同期処理の違いを、実際の動作で確認しましょう。

同期vs非同期の違い

同期処理は次の行に進む前に完了まで待ちます。
非同期処理は完了を待たずに次の行へ進みます。
「同期実行」と「非同期実行」ボタンでログの順番の違いを見てみましょう。

同期実行 非同期実行
    setTimeout による非同期のサンプルコード
    // 非同期: setTimeout はすぐに返り、コールバックは後で実行される
console.log("1. 処理開始");

setTimeout(() => {
  console.log("3. タイムアウト後（1秒後）");
}, 1000);

console.log("2. タイムアウトをセット後、すぐに次の行へ進む");

// 出力順:
// 1. 処理開始
// 2. タイムアウトをセット後、すぐに次の行へ進む
// 3. タイムアウト後（1秒後）

    コールバックとは

    コールバックは「処理が終わったら呼んでね」と渡す関数です。

    コールバック実行
      コールバックのサンプルコード
      function fetchData(callback) {
  setTimeout(() => {
    const data = { user: "田中", score: 92 };
    callback(data); // 完了したらコールバックを呼ぶ
  }, 1000);
}

fetchData((result) => {
  console.log("受け取ったデータ:", result.user);
});
